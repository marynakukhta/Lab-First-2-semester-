Use Case Diagram:
•	Назви Use Cases є доволі зрозумілими, в глосарії описані всі важливі поняття.
•	Діаграма включає не всі важливі сценарії соц. мережі.(можливість поставити лайк, переглянути відео, функціональність сторінки користувача не повна).
•	Не вказані типи зв’язку між деякими use cases.
•	Було б не зайвим додати коментарі до Use Cases які включають обширу функціональність.(View user page, sittings, make post)
Загалом можна зрозуміти яку функціональність матиме користувач того чи іншого типу.(Guest/abstract/registered )

Object diagram:
•	Відповідає архітектурі Class Diagram
•	Створені об'єкти відповідних класів та показані зв'язки між ними.

Class Diagram:
•	Відображені всі можливості з Use Case diagram.
•	Через неповноту глосарія важко зрозуміти деякі аспекти.
•	Взаємодія між класами побудована добре.
